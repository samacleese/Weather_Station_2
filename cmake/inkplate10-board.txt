# ABOUTME: Custom board definition for the Inkplate 10, layered onto the esp32:esp32 core.
# ABOUTME: Appended to the installed esp32:esp32 package's boards.txt by the Containerfile.
#
# Croduino_Boards:Inkplate is unmaintained (last board release 1.0.1, package repo untouched
# since 2022) and only ever bundled ESP32 Arduino 1.0.5-rc2, too old for current InkplateLibrary
# releases. This file adds an "Inkplate10" board entry directly under the actively maintained
# esp32:esp32 core instead, reusing its platform.txt, toolchain, and the stock "esp32" variant
# (InkplateLibrary hardcodes its own GPIO pins in src/boards/Inkplate10/pins.h rather than
# sourcing them from pins_arduino.h, so the generic esp32 variant is sufficient).
#
# Derived by downloading the esp32:esp32@3.3.10 platform archive directly
# (https://github.com/espressif/arduino-esp32/releases/download/3.3.10/esp32-core-3.3.10.zip)
# and copying its "ESP32 Dev Module" entry (id "esp32", boards.txt lines ~1811-2007), renaming
# the id to "Inkplate10" and setting build.board=INKPLATE10 so the toolchain defines
# -DARDUINO_INKPLATE10, which is what InkplateLibrary branches on. Menu options were trimmed to
# those actually used or useful here (dropped JTAGAdapter, LoopCore, EventsCore, ZigbeeMode).
# EraseFlash was added (not present upstream) to make a full chip erase a one-click board option
# when first flashing this core, since the old core used different bootloader/partition offsets.
#
# To regenerate against a newer esp32:esp32 release: download that release's platform archive,
# diff its "esp32." stanza against this file, and re-apply the same rename + trim.
